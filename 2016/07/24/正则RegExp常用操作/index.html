正则RegExp常用操作

发表于 2016-07-24   |  

网上有太多太多关于正则的文章，写的非常详细，且篇幅很长。但是对于多数人来说其实是没有精力看完并全部吸收掉的。本文旨在用简洁的语言囊括日常工作中需要的多数用法，如果您想进行全面的了解，建议去大神阮一峰的博客学习。

一、RegExp的创建

  • var reg = /xyz/; var reg2 = /xyz/i; //在编译时新建正则表达式
  • var reg = new RegExp(‘xyz’); var reg2 = new RegExp(‘’xyz,”i”); //在运行时新建表达式
    备注：这两种方法等价，通常会使用第一种字面量形式创建。

二、RegExp的属性

修饰类相关，返回一个布尔值，表示对应的修饰符是否设置

三、RegExp的方法

  1. test()
    正则对象的test方法返回一个布尔值，表示当前模式是否能匹配参数字符串

   注： 如果正则表达式带有g修饰符，则每一次test方法都从上一次结束的位置开始向后匹配。

四、其他

4.1 匹配规则

4.11 点字符(.)
匹配除回车(\r)、换行(\n)、行分隔符(\u2028)和段分隔符(\u2029)以外的所有字符。
4.12 选择符(|)
竖线表示选择符or
4.13 位置字符
位置字符用来提示字符所处位置，主要有两个字符。^表示字符串开始的位置，$表示字符串结束的位置。

4.14 转义符
正则模式中，需要用到斜杠转义的，共有12个字符：^ . [ $ ( ) | * + ? { \

需特别注意的是，若使用JS方法生成对象，转义需要两个斜杠，因为字符内部会先转义一次。

4.2 字符类

字符类(class)表示有一系列字符可供选择，只要匹配其中一个就可以，所有可供选择的字符都放在方括号内。

4.21 脱字符(^)
如果方括号的第一个字符是^，则表示除了字符类中的字符，其他字符都可以匹配。

如果方括号内只有^，没有其他字符，则表示匹配一切字符，而点号(.)是不包括换行符的。(脱字符只有在字符类的第一个位置才有意义，其他位置只代表字面的含义)

4.22 连字符(-)
连字符用来提供简写形式，表示字符的连续范围。(需写在方括号中，才具有简写的意义)

4.3 预定义模式

预定义模式指的是某些常见模式的简写方式。

\d 匹配0到9之间的任一数字，相当于[0-9]
\D 匹配所有0到9以外的字符，相当于[^0-9]
\w 匹配任意的字母、数字、下划线，相当于[a-zA-Z0-9_]
\W 匹配所有字母、数字、下划线以外的字符，相当于[^a-zA-Z0-9_]
\s 匹配空格(包括制表符、空格符、断行符等)，相当于[\t\r\n\v\f]
\S 匹配非空格的字符，相当于[^\t\r\n\v\f]
4.4 其他

4.41 重复类
模式的精确匹配次数，使用大括号表示。{n}表示恰好重复n次，{n,m}表示不少于n次，不多于m次。
4.42 量词类


? 表示某个模式出现0次或1次，等同于{0,1}

  • 星号表示某个模式出现0次或多次，等同于{0,}
  • 加号表示某个模式出现1次或多次，等同于{1,}

    五、字符串对象的方法

    5.1 String.prototype.match()
    对字符串进行字符匹配，返回匹配结果。
5.2 String.prototype.search()

返回第一个满足匹配结果在字符串中的位置，如若没有任何匹配，则返回-1，该方法会忽略g修饰符。

5.3 String.prototype.replace()

替换匹配的值，接受两个参数，第一个是搜索模式，第二个是替换的内容。

replace()方法的第二个参数还可以是个函数，将每一个匹配内容替换为函数返回值。
